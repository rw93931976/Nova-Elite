The red-screen fatal boot error ("Cannot access 'Yt' before initialization" at `index-CL_v2GMG.js:78:17007` and down the call stack) is almost certainly caused by a **circular dependency** (or a very tricky initialization order) that only breaks in production builds.

Vite's dev server forgives a lot of circular imports, but Rollup (the production bundler) linearizes modules strictly → minification + chunk splitting can push a `const`/`let` access before its declaration in the final execution order → Temporal Dead Zone (TDZ) crash with an obfuscated name like 'Yt'.

This matches hundreds of similar reports on GitHub/Vite issues/Stack Overflow from 2023–2025.

2. **Hunt circular dependencies (most likely fix)**
   Install a detector temporarily:
   ```bash
   npm install --save-dev madge
   ```
   Run:
   ```bash
   npx madge --circular --extensions ts,tsx,js,jsx src/
   ```
   → Look for red lines showing loops (e.g. `useNova.ts → NovaCore.ts → ReasoningEngine.ts → back to useNova` or similar).

   Common culprits in your codebase:
   - `useNova.ts` imports `NovaCore` and `ConnectionManager`
   - `NovaCore` likely imports agents like `ReasoningEngine`
   - `ReasoningEngine` imports `EmotionAgent`, `SchoolingAgent`, `MemoryAgent` — check if any of those reach back (directly or via shared utils/types)
   - Any barrel file (`index.ts` re-exporting components/types) that creates import cycles
   - Hooks/utils that import components or vice-versa

   **Quick fixes when you find a cycle:**
   - Move shared logic (types, constants, small utils) to a neutral file both can import.
   - Use **lazy/dynamic imports** for the cyclic part:
     ```ts
     // Instead of: import { Something } from './Something';
     const Something = lazy(() => import('./Something'));
     ```
   - Or defer usage inside `useEffect` / callback.
